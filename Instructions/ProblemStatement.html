Birthday cards application:

Create an application using Node.js and MongoDB that help to manage birthdays.

  • birthday-cards is the name of the database used in the application.

  • There is a single collection called birthdays inside the database birthday-cards.

ROUTERS:

There is a single file for routers namely birthdays.js that reside inside src/routers. The endpoints and their functionalities are as follows.

1) /birthdays -> POST Method -> This route should save the data sent with the request body inside the birthdays collection.

Sample data sent with the request: 

{
    "name": "person_one",
    "dob": "1994-03-23T00:00:00.000Z",
    "relationship": "Brother"
}

  • If the insertion was successful, then you should send a response code of 201.

  • If some data validations failed and the data is not inserted into the collection then you should send a response code of 400.

2) /birthdays-> GET Method -> This route should fetch the data from the birthdays collection. This route also has three optional query parameters namely name, dob and type.

  • If no query parameters were sent with the request URL, then you should fetch all the data from the birthdays collection.

  • If the parameter name is sent in the URL with the value asc, then you should sort the data based on the alphabetical order of the names(A-Z) and if desc is sent as the value then you should sort the data based on reversed alphabetical order(Z-A).

  • If the parameter dob is sent in the URL with the value asc, then you should sort the data in ascending order of the dob and if desc is sent as the value then you should sort the data in descending order of the dob.

  • If the parameter type is sent with the URL, then you should fetch the data that are having relationship equal to the value of type.

  • If the parameters type and name with the value asc are sent with the request URL then you should sort the data based on the alphabetical order of the names(A-Z) and if desc is sent as the value then you should sort the data based on reversed alphabetical order(Z-A) and fetch the data that are having relationship equal to the value of type.

  • If the parameters type and dob with the value asc are sent with the request URL then you should sort the data in ascending order of the dob and if desc is sent as the value then you should sort the data in descending order of the dob and fetch the data that are having relationship equal to the value of type.

  • If the data is fetched successfully from the database, then you should send a response code of 200.

  • If something went wrong and the data is not fetched successfully from the database, then you should send a response code of 400.

MongoDB Commands:

  • You can open the mongo shell by running mongo from the terminal.

  • You can view all the data from the database in MongoDB by running show dbs from the mongo shell.

  • You can select the database by running use birthday-cards.

  • You can view the names of the collections by running show collections.

  • You can view the data inside a collection by running db.birthdays.find()

  • Press ctrl+c to exit.
